## Selected Quick Win: Pull-to-Refresh Component
**Why?** High-leverage improvement for mobile UX:
1. Addresses latency tolerance - gives users clear way to refresh stale data
2. Native mobile pattern users expect
3. Integrates with existing Capacitor haptics plugin
4. Works with React Query's refetch pattern
5. Improves perceived performance on slow networks

## Testing Instructions

### Manual Testing (Mobile/Simulator)
1. Open the app on a mobile device or iOS/Android simulator
2. Navigate to the Dashboard
3. Scroll to the top of the page
4. Pull down with finger - you should see:
   - An arrow indicator that rotates as you pull
   - Haptic feedback when threshold is reached (native only)
   - The arrow changes to primary color when ready to release
   - On release, a spinner shows and data refreshes
5. Verify that stats update after refresh completes
6. Test on desktop - pull-to-refresh should be disabled

### Edge Cases to Test
- Pull when already scrolled down (should not trigger)
- Pull up (should not trigger)
- Pull partially and release (should cancel, not refresh)
- Network offline during refresh (should handle gracefully)

## Notes
- Component only activates on mobile (uses useIsMobile hook)
- Threshold of 80px provides good balance of intentional vs accidental triggers
- 0.5 resistance factor gives natural iOS-like feel
- Haptics use ImpactStyle.Medium for noticeable but not jarring feedback
